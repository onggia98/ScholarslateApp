3. JWT Auth Flow — Đăng ký / Đăng nhập / Xác thực
Toàn bộ vòng đời JWT: từ register → nhận token → gắn vào request → xác thực → controller nhận userId.

Sequence Diagram: Register & Login

→ register() existsByEmail? SELECT email

Sau khi nhận token: Frontend lưu vào localStorage.token. Mọi request tiếp theo gắn Authorization: Bearer <token>. JwtAuthFilter decode → lấy userId + role → set SecurityContext → Controller nhận qua @AuthenticationPrincipal UUID userId.

JWT Token Structure & Flow

JWT Claims (payload)
{
  "sub": "uuid-user-id",
  "email": "user@mail.com",
  "role": "USER",
  "iat": 1716800000,
  "exp": 1716886400 // +24h
}
Ký bằng HMAC-SHA256 với JWT_SECRET (≥256 bit). Không có refresh token — logout bằng xóa localStorage.

7. Scheduler Flow — Background Processing
Hai scheduler hoạt động song song trên thread pool riêng (pool size=3). MainScheduler chạy theo cron, RetryScheduler chạy fixedDelay.

Timeline & Coordination giữa 2 Schedulers

Tại sao pool size = 3? MainScheduler + RetryScheduler = 2 threads đang chạy đồng thời. Thread thứ 3 là buffer cho manual trigger từ admin (chạy qua Thread.ofVirtual() — không dùng pool). Nếu pool size = 1 → MainScheduler block RetryScheduler → retry không chạy trong lúc Main đang fetch.
