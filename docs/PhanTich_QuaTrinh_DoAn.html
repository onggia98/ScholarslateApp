1. Bài toán & Mục tiêu

Vấn đề thực tế

Số lượng bài báo khoa học trên arXiv tăng theo cấp số nhân — hàng nghìn bài mỗi ngày. Nhà nghiên cứu và sinh viên khó theo kịp những công trình mới nhất trong lĩnh vực của mình, phải thủ công tìm kiếm và đọc lọc từng bài.

Vấn đề

Không có công cụ tự động theo dõi, tóm tắt, và gợi ý paper phù hợp với từng chủ đề nghiên cứu cá nhân.

Giải pháp

ScholarSlate tự động fetch paper từ arXiv theo keywords, dùng AI để tóm tắt, tính chất lượng, và gợi ý paper liên quan qua vector similarity.

7. Pipeline AI — Thiết kế chi tiết

Luồng xử lý chính (Main Pipeline — 06:00 hàng ngày)

1. Lấy Active Topics & keywords
Query is_active = true; dedup keywords bằng LinkedHashSet để tránh fetch cùng keyword 2 lần
2. Fetch từ arXiv API
Delay 1500ms giữa requests (phát hiện 429 ở 350ms trong production). Max 10 paper/keyword. URL bắt buộc sortBy=submittedDate&sortOrder=descending
3. Upsert paper (ON CONFLICT arxiv_id DO NOTHING)
Paper mới: PENDING, retry_count=0. Paper cũ bị skip idempotently
4. Embedding (HuggingFace — BAAI/bge-small-en-v1.5)
Batch max 32. HTTP 503 (model cold start): chờ 30s, retry 1 lần
5. Duplicate Detection (pgvector cosine)
Cosine distance < 0.05 (similarity > 0.95) trong window 90 ngày. Duplicate → is_duplicate=true, DONE
6. AI Summary (Groq — llama-3.1-8b-instant)
Single call với response_format: json_object. Validate: quality_score ∈ [0,10], summary ≤ 2000 chars. Delay 2s giữa các call
7. Topic Matching (phraseto_tsquery + GIN)
phraseto_tsquery (không phải plainto) để match chính xác cụm keyword theo thứ tự
8. Notification Creation
ON CONFLICT DO NOTHING — đảm bảo mỗi (user, paper, type) chỉ có 1 notification

Transaction design

// Mỗi paper chạy trong transaction độc lập
@Transactional(propagation = Propagation.REQUIRES_NEW)
public boolean process(Paper paper) { ... }

// Outer orchestrator KHÔNG có @Transactional
// → tránh exhaustion HikariCP pool (max 20 connections)
public void runDailyFetch() { ... }

Retry Scheduler (mỗi 30 phút)

Query: processing_status = 'FAILED' AND retry_count < 3

Retry pipeline AI (embed → dup check → Groq). Cập nhật last_retry_at = NOW(). Max 3 lần — sau đó giữ FAILED, admin can thiệp qua UC17.

Lý do chọn BAAI/bge-small-en-v1.5

Vấn đề phát hiện trong quá trình làm Ban đầu thiết kế dùng sentence-transformers/all-MiniLM-L6-v2, nhưng HuggingFace phân loại model này là sentence-similarity pipeline, không phải feature-extraction → gọi API bị lỗi. Chuyển sang BAAI/bge-small-en-v1.5 — feature-extraction pipeline, cùng 384 dims, không cần thay đổi schema.

8. Vector Search — pgvector

Custom VectorUserType

Vì hypersistence-utils không tương thích Spring Boot 4.x / Hibernate 7.x, nhóm implement thủ công VectorUserType implements UserType<float[]>:

// Serialize: float[] → PGobject kiểu "vector"
// Deserialize: PGobject → float[]
// Đăng ký qua @TypeDef ở entity Paper hoặc package-info.java

Recommendation query

-- ef_search=128 set global qua HikariCP connection-init-sql
SELECT * FROM paper
WHERE processing_status = 'DONE'
  AND is_duplicate = false
  AND id <> :paperId
  AND published_at >= NOW() - INTERVAL '1 year'
  AND embedding <=> CAST(:embedding AS vector) < 0.5
ORDER BY embedding <=> CAST(:embedding AS vector)
LIMIT 10

Duplicate detection query

SELECT * FROM paper
WHERE id <> :paperId
  AND is_duplicate = false
  AND processing_status = 'DONE'
  AND published_at >= NOW() - INTERVAL '90 days'
  AND embedding <=> CAST(:embedding AS vector) < 0.05
ORDER BY embedding <=> CAST(:embedding AS vector)
LIMIT 1

Caching recommendation

Spring Cache + Caffeine: key = paper_id, TTL = 1 giờ, max 500 entries.

Cache miss → query HNSW. Cache hit → trả ngay, không tốn DB roundtrip.

connection-init-sql: "SET hnsw.ef_search = 128" — set global cho toàn bộ pool, không cần SET LOCAL trong từng query.

Những gì học được

Spring Boot 4.x có nhiều breaking changes so với 3.x
MapStruct, hypersistence-utils, Flyway config — cần kiểm tra compatibility trước khi chọn dependency
Rate limits của AI APIs rất thực tế
HuggingFace cold start (20–30s), Groq 30 RPM free tier, arXiv 429 — phải thiết kế retry và delay từ đầu
Transaction design quan trọng hơn tưởng
REQUIRES_NEW per paper + outer không có @Transactional — tránh connection pool exhaustion và isolation lỗi
Full-text search: plainto vs phraseto
phraseto_tsquery cho topic matching (cụm từ cần đúng thứ tự), plainto_tsquery cho search tổng quát
Production environment khác local nhiều hơn tưởng
iOS Safari TLD, Railway env vars, Vercel SPA routing, Flyway datasource config — chỉ phát hiện khi deploy thật
